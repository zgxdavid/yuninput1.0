# 候选窗多显示器回归清单

更新时间：2026-04-10

## 1. 目标

验证候选窗在多显示器环境下，能够稳定跟随输入锚点（插入符/光标），并始终约束在正确显示器的工作区内，不出现跨屏抖动、跳屏或越界。

## 2. 适用版本与关注点

- 适用范围：含 2026-03-26 候选窗工作区选屏修复，以及 2026-04-10 候选窗位置吸附与双缓冲绘制修复的构建。
- 变更关注点：工作区选择优先基于输入锚点，而不是窗口偏移后的左上角。
- 变更关注点：快速输入、翻页或选中项切换时，候选窗不应出现明显闪烁、宽度回摆或 1 到 3 像素级连续抖动。

## 3. 测试前准备

1. 在 Windows 显示设置中准备至少 2 块显示器。
2. 覆盖以下屏幕排布之一：
   - 副屏在主屏右侧
   - 副屏在主屏左侧
   - 副屏在主屏上方或下方
3. 将每块屏幕缩放比例设置为常见组合之一（例如 100% + 125%）。
4. 启用匀码输入法，并确认可正常弹出候选窗。

## 4. 基础回归步骤

1. 在主屏文本框中输入编码，观察候选窗是否贴近插入符，且完整显示在主屏可用区域内。
2. 在副屏文本框中输入编码，观察候选窗是否贴近插入符，且完整显示在副屏可用区域内。
3. 在两块屏幕之间来回切换焦点输入，观察候选窗是否稳定切屏、无明显迟滞和抖动。
4. 在任务栏附近、屏幕边缘和角落输入，观察候选窗是否自动避让并保持可见。

## 5. 边界场景清单

1. 插入符位于屏幕左右边界附近：候选窗不应跳到另一块屏幕。
2. 插入符位于屏幕上下边界附近：候选窗应在上/下方合理选择位置并保持完整可见。
3. 快速连续输入并翻页（-、=、PgUp、PgDn）：候选窗位置不应频繁闪跳。
4. 从有锚点应用切到锚点信息较弱的应用（仅能回退到光标）：候选窗仍应在当前交互屏幕内。
5. 应用窗口跨屏拖动后继续输入：候选窗应跟随当前实际输入位置而非停留旧屏幕。
6. 连续上下移动选中项或快速删改编码：候选窗不应因高亮宽度变化而出现明显宽度抖动。

## 6. 判定标准

- 通过：
  - 候选窗始终出现在当前输入锚点所在显示器。
  - 候选窗始终位于该显示器工作区内，不遮挡到不可见区域。
  - 焦点跨屏切换时，候选窗切换稳定、无连续跳屏。
  - 快速输入和高频重绘时，候选窗无肉眼明显闪烁，宽度和位置变化平滑。
- 不通过：
  - 候选窗在边界附近错误出现在相邻显示器。
  - 候选窗部分超出当前显示器工作区，出现截断。
  - 输入过程中出现明显抖动、来回跳屏或长时间停留在错误屏幕。
  - 选中项切换或快速输入时出现持续闪白、宽度来回收放或细碎抖动。

## 7. 失败时记录模板

建议记录以下信息，便于复现与定位：

1. 屏幕布局（左右/上下关系、主副屏、缩放比例）。
2. 失败应用与输入控件类型（例如记事本、浏览器输入框、IDE 编辑器）。
3. 触发步骤（最小复现路径）。
4. 实际现象（跳到哪块屏、是否越界、是否抖动）。
5. 对应日志与版本标识：
   - `%LOCALAPPDATA%\yuninput\runtime.log`
   - 当前部署 DLL 路径与 build marker

可直接使用仓库模板文件逐条记录：

- `docs/multi-monitor-candidate-window-test-log-template.tsv`
- `docs/multi-monitor-candidate-window-test-log-template.csv`

建议填写规范：

- `Result`：`PASS` / `FAIL` / `BLOCKED`
- `Notes`：记录偏差现象、截图文件名或关键日志片段
- `IssueID`：缺陷单号或复现编号（例如 `MM-BUG-20260326-01`）
- `ReproStepsRef`：复现材料路径（例如 `diagnostics/...`、截图文件名或工单链接）

## 8. 推荐最小回归矩阵

每次改动候选窗定位逻辑后，至少执行：

1. 双屏左右排布 + 主屏输入 + 副屏输入。
2. 双屏上下排布 + 边缘输入（四角各一次）。
3. 混合缩放（100% + 125%）下的跨屏来回输入。
