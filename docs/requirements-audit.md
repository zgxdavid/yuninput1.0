## 已完成

### 1. TSF 基础骨架与构建链路

- 已具备 C++ DLL 工程、TSF Text Service、COM 注册/反注册、Release 构建脚本。
- 安装、注册、卸载、清理脚本均已存在，且可构建通过。

当前判断：已完成。

### 2. 基础候选交互

- 候选窗显示、跟随光标、边界约束已实现。
- 数字键选词、空格上屏、回车优先精确候选、Esc 清空、Backspace 回退已实现。
- 翻页键支持 `-` / `=`，兼容 `[` / `]`，并支持 `PgUp` / `PgDn`。
- `Tab` / `Shift+Tab` 前后导航已实现。

当前判断：已完成。

### 3. 用户控制闭环

- `Ctrl+1-9` 置顶候选已实现。
- `Ctrl+Delete` 屏蔽当前高亮候选已实现。
- 2026-03-23 已修正为优先按当前输入编码生效，和 README 的描述一致。

当前判断：已完成。

### 4. 基础学习能力

- 上屏调频已实现并持久化。
- 上下文关联学习已实现并持久化。
- 最近上屏后的下一段编码联想已接入排序。
- 连续上屏和保留余码已实现。

当前判断：已完成，但算法质量仍需继续优化。

### 5. 配置工具与常用入口

- 配置工具已存在，可修改候选页大小、自动顶屏、Tab 导航、空码提示音、回车优先策略、上下文关联配置等。
- `Ctrl+F2` 可打开配置工具。
- `F2` / `APPS` 可打开状态菜单。
- 语言栏按钮已实现，右键菜单可打开配置、系统设置和运行日志。

当前判断：已完成。

### 6. 外部码表导入链路

- 已支持通过 `scripts/import_table_dict.ps1` 导入 Fcitx / IBus Table 类文本码表。
- 已支持从 `data/` 目录批量加载多个 `.dict` 文件。
- 安装时复制整个 `data/` 目录下的词库文件，而不是只复制单个示例词库。

当前判断：已完成。

## 部分完成

### 1. “类似郑码输入体验”

- 当前已具备“郑码表驱动”的运行方式，可以加载现成郑码码表并进行候选查询。
- 候选排序已经往形码习惯靠拢：优先精确码，再看用户频率、静态分、码长和载入顺序。
- 但这仍然主要依赖外部码表，不是完整的郑码规则实现。

当前判断：部分完成。

### 2. 候选质量与输入法手感

- 已有候选预选、上下文关联、用户置顶、连续上屏等增强。
- 但“有没有达到你的要求”不能只看功能点存在，还要看排序质量、常用字命中率、重码习惯、短码与全码关系是否自然。
- 这些目前没有看到系统性的验证矩阵，也没有证据表明已达到成熟输入法水准。

当前判断：部分完成。

### 3. 系统集成体验

- 已接入语言栏按钮、状态菜单、图标资源、Description/IconFile/IconIndex 注册写入。
- 但系统外壳是否稳定展示自定义名称/图标，仍然存在现场争议，不能直接判定为彻底解决。

当前判断：部分完成。

## 有争议

### 1. 任务栏输入法名称与图标是否真正修复

- 诊断快照显示 HKCU/HKLM 的规范值已经写入：
  - Description 为“匀码输入法”
  - IconFile 已指向安装目录 DLL
  - IconIndex 为 0
- 但同一份诊断里实际枚举到的注册表输出仍出现乱码形式的 Description。
- 这说明“值写进去了”与“系统外壳稳定显示正确”之间仍有差距，至少不能判定为完全闭环。

当前判断：有争议。

### 2. 当前功能是否已经达到“你的要求”

- 如果目标只是“先做出一个可安装、可输入、可加载郑码表、带基础候选和学习能力的原型”，那已经超过纯骨架阶段。
- 如果目标是“做成真正可长期使用、接近成熟郑码体验的输入法”，那还远未达到要求。

当前判断：有争议，取决于你的目标层级。

## 当前总判断

### 已经有的东西

- 可构建、可安装、可注册的 TSF 输入法外壳
- 候选窗与一整套基础输入交互
- 用户置顶、删词屏蔽、调频、上下文学习、配置工具
- 外部郑码码表导入能力

### 仍然缺的核心东西

- 规则驱动的郑码拆根与取码
- 自动构词码
- 接近成熟产品的排序与词组策略
- 更完整的系统级 TSF UI 集成与稳定性闭环

## 建议作为下一阶段的硬目标

如果后续要继续开发，建议优先把以下目标作为真正的“主线需求”，而不是继续堆外围功能：

1. 明确“目标是表驱动郑码原型”还是“规则驱动郑码输入法”。
2. 若目标是后者，优先补郑码规则层，而不是继续补外围菜单和安装细节。
3. 对常用单字、词组、重码顺序建立一套最小可验证样例，用它来判断输入体验是否真的达标。
4. 把任务栏名称/图标问题单独作为系统集成问题跟进，不与算法层混在一起。
