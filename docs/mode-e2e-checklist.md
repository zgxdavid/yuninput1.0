# 两模式端到端验收清单（10组编码）

更新时间：2026-03-23

## 验收目标

验证以下两种输入模式在同一编码下能够加载不同词库并给出符合预期的候选结果：

- `zhengma-large`（大字符集郑码）
- `zhengma-large-pinyin`（拼音单字输入，候选右侧显示郑码提示）

说明：仓库仍保留 `data/zhengma.dict` 作为导入与对照资料，但当前运行时与配置工具只暴露以上两种模式。

## 代码路径确认

- 运行时按 `dictionary_profile` 选择词库：`src/TextService.cpp`
- 配置工具支持两模式下拉项并读写 `dictionary_profile`：`tools/YuninputConfig.cs`
- 当前运行模式词库文件：`data/zhengma-large.dict`、`data/zhengma-pinyin.dict`

## 执行步骤（人工可复现）

1. 打开配置工具，将模式切换到目标值（`zhengma-large` / `zhengma-large-pinyin`）。
2. 重新激活输入法（建议切换输入法一次，必要时重启目标应用）。
3. 逐条输入下表中的编码，记录第1候选（必要时记录前8候选）。
4. 两种模式都跑完后，对照“期望差异”列判断是否符合。

## 结论口径

- 本轮已确认：两模式词库文件齐备、配置项齐备、运行时代码分支齐备。
- 本表可作为“模式切换是否生效”的最小回归集。
- 若人工实测中任一编码在两模式下完全同候选，优先排查：
  - `settings.json` 中 `dictionary_profile` 是否真的切换
  - 输入法是否重新激活
  - 部署目录 `data` 下两份在用词库是否为最新版本
