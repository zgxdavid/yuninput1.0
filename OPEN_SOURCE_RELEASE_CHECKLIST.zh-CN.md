# 开源发布检查清单

本清单用于在将 yuninput 发布到 GitHub 之前，检查代码、词库、安装包和声明文件是否满足最基本的开源与分发要求。

## 一、权属与审批

1. 确认本仓库原创代码可由 Intel 对外开源发布。
2. 确认已经满足团队、法务、管理链路或内部流程要求。
3. 确认仓库中不存在不允许对外公开的内部文档、账号信息、邮箱、路径或调试痕迹。

## 二、许可证文件

1. 确认仓库根目录包含 `LICENSE`。
2. 确认 `LICENSE` 中的版权人和许可证类型与实际发布策略一致。
3. 确认 `NOTICE.md` 与 `NOTICE.zh-CN.md` 中的 Intel 权属声明准确无误。

## 三、第三方材料

1. 逐项检查 `third_party/`、`data/`、`tools/`、`scripts/` 中是否包含第三方材料。
2. 对每一项第三方材料记录：来源链接、原始项目名、许可证、是否允许再分发。
3. 对许可不明、来源不明或不允许再分发的材料，移出仓库或替换为可分发版本。
4. 更新 `THIRD_PARTY_NOTICES.md` 与 `THIRD_PARTY_NOTICES.zh-CN.md`。

## 四、Fcitx 相关核验

1. 核对 `third_party/fcitx/zhengma.txt` 的上游来源与许可。
2. 核对 `third_party/fcitx/zhengma-large.txt` 的上游来源与许可。
3. 核对 `third_party/fcitx/zhengma-pinyin.txt` 的上游来源与许可。
4. 如果无法核实其中任一文件的上游许可，则不要公开分发该文件及其直接派生产物。
5. 如继续保留这些文件，必须在声明文件中补齐准确上游链接和许可文本。

## 五、郑码相关声明

1. 确认 `NOTICE.md`、`NOTICE.zh-CN.md`、`THIRD_PARTY_NOTICES.md` 中均保留郑码权利归属声明。
2. 确认对外说明未暗示本项目拥有郑码相关知识产权。
3. 如产品名称、介绍文案或发布页使用“郑码”字样，确认其表述仅用于兼容、说明或事实描述。

## 六、代码与仓库内容清理

1. 检查是否包含临时日志、个人路径、调试快照、安装日志或本机生成文件。
2. 检查 `diagnostics/` 下内容是否适合公开；如不适合，应移除或迁出历史仓库。
3. 检查安装脚本、注册脚本和打包脚本中是否包含仅适用于内部环境的路径或配置。
4. 检查 README、docs、注释中是否包含不应公开的内部信息。

## 七、发布产物

1. 确认发布的源码包、EXE、MSI 与仓库声明一致。
2. 确认安装包内附带必要的许可证与声明文件。
3. 确认安装包未额外包含许可不明的词库、字库、图标或其他资源。

## 八、GitHub 发布页

1. 在仓库首页说明项目用途、当前阶段和适用范围。
2. 在发布说明中列出许可证、第三方来源说明和已知限制。
3. 如某些词库因许可原因不能附带，明确说明用户需自行导入。

## 九、最终发布前确认

1. 再次运行一次全量构建与安装验证。
2. 重新生成发布包并记录哈希。
3. 由维护者逐项确认本清单后，再执行公开发布。
